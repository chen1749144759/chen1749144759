1.1 链式操作思想

  • 链式操作是指对一个对象中的函数进行操作后反悔的结果是这个对象,以便后续可以继续调用该对象的其他方法, 例如:
//定义一个JS类 
function Demo() { 
} 
//扩展它的元素 
Demo.prototype ={ 
setName:function (name) { 
this.name = name; 
return this;  //返回该对象
}, 
getName:function () { 
return this.name; 
}, 
setAge:function (age) { 
this.age = age; 
return this;  //返回该对象
} 
}; 
////工厂函数 
function D() { 
return new Demo(); 
} 
//去实现可链式的调用 
D().setName("CJ").setAge(18).setName();

1.2 $(function..) 与 window.onload()的区别

  • 前者在网页中DOM节点加载完毕就会执行,并可以执行多次
  • 后者会在网页完全加载完毕执行,并只会执行最后一个函数[^注解1.2]
//两者区别
//前者 
$(function(){
    alert(1);
});
$(function(){
    alert(2);
});
//两次都会执行 

//后者
window.onload = function(){
    alert(1);
}

window.onload = function(){
    alert(2);
}
//只会执行最后一次 结果 2

1.3 DOM对象和 jQuery对象的区别

  • 同样一段代码 两种不同的获取方式 例:

    <input id="name" type="text" />
<script>
    //DOM 
    var name = document.getElementById("name").value;
    //jQuery
    var $name = $("#name").val();

    //比起DOM对象复杂难记的函数方法,你只需记住jQuery中的 "#" "." 等等
    //实际工作中也常常只用到 Id选择器和 类选择器  详细选择器类型下面会有记录
</script>

1.4 选择器

注意特殊字符的转义

例如:

//如果标签中有 " # [] ()  则需要加入'\\'进行转义,如下: "
<input id="name[1]" type="text" />
<script>
    $("#name\\[1\\]").css(...);
</script>
  • jQuery一共给我们提供了9种选择器 他们分别是:

1.4.1. 基本选择器

$("#id") //id选择器 返回单个元素
$(".class") //类选择器 返回集合元素
$("element") //element翻译过来就是元素/标签  返回所有符合的标签
$("*") //匹配所有元素 
$("div,span,p.myClass") //返回所有 div,span以及p标签中class为MyClass的标签

1.4.2. 层次选择器

[^注解1.4.1] [^注解1.4.2]

$("div sapn") //选取div标签下所有span元素 不论层次 参见 注解1.4.1
$("div > sapn") //只会选择父标签是div的span标签 
$("div + span") //选取紧跟在div标签后的span元素 参见 注解 1.4.2 
$("div ~ span") //选取div后面的所有span兄弟元素 
//jQuery API 中也提供一些类似功能的函数 .next()  .nextAll()

1.4.3. 过滤选择器

$("div:first") //选取第一个div元素
$("div:last") //选取最后一个div元素
$("div:not(.class)") //选取所有类名不是 class的div标签
$("div > li:eq(1)") //选取div元素下第二个li元素 
$("div:even") // 选取偶数的div元素  index从0开始
$("div:odd") //选取奇数的div元素 index从0开始
$("div:lt(index)") // 选取索引小于index的元素
$(":header") //选取网页中所有的标题元素 h1 h2 ...
$("div:animated") // 选取当前正在执行动画的元素
$(":focus") //选取当前获得焦点的元素

1.4.4. 内容过滤选择器

$("div:contains('我')") // 选取内容含有 '我' 的div元素
$("div:empty") // 选取没有子元素或内容的 '空标签'  反而子标签和内容文本只要包含一个就不会被选中
$("div:parent") // 选取含有子元素或者内容的标签  与选取没有的相反
$("div:has(span)") //选择含有span标签的div标签 无视层级关系

1.4.5. 可见性过滤选择器

$(":hidden") //选取所有不可见元素
$(":visible") //选取所有可见元素

1.4.6. 属性过滤选择器

[^注解1.4.3]

$("div[class]") // 选择拥有属性 class 的div元素
$("div[class=myClass]") //选择class名为myClass 的div元素
$("div[class!=myClass]") // 选择class不等于myClass的div元素
$("div[class^=my]") // 选择class名称以 my开头的div元素 
$("div[class$=Class]") //选择class名称以 Class结尾的div元素
$("div[class*=yCl]") //选择class名称中有"yCl"字符 的div元素
$("div[class|=my]") // 选择class名称等于 my 或者以my开头后面跟一个连字符的div元素 参见 注解1.4.3
$("div[class~=class]") //选择用空格分隔的值中包含class 的div元素
//下面这个选择器为上述选择器组合 代码只组合两个为示例
$("div[title][class*=yCl]") // 这行代码表示: 选择有title属性并且class中含有字符'yCl'的div标签

1.4.7 子元素过滤选择器

  • 对比子元素选择器之前的一些选择器,html中的DOM结构会有一些不同;
  • 下面我将html网页中的结构输出如下,方便理解:
<body>
    <ul>1
        <li>1.1</li>
        <li>1.2</li>
        <li>1.3</li>
    </ul>

    <ul>2
        <li>2.1</li>
        <li>2.2</li>
        <li>2.3</li>
    </ul>

    <ul>3
        <li>3.1</li>
    </ul>
</body>
  • 对照上图结构理解选择器

–多种写法,不多演示–

$("body ul:nth-child(2n)") // 选取父元素body 下索引是2或者2的倍数的 ul元素("n"代表本身及其倍数 , nth-child 的index从1开始)
$("ul > li:first-child") // 选取父元素是ul 的 第一个li元素
$("ul > li:last-child") // 选取父元素是ul 的 最后一个li元素
$("ul li:only-child") // 选取父元素是 ul 且只有一个子元素并且还是li元素 的子元素

1.4.8 表单对象属性过滤选择器

  • 对比其他选择器,表单对象属性过滤选择器会在一个相对稳定的结构中对文本进行操作

  • 将带有from的表单输出如下:

<body>
<form id="form1" action="#">
    <input disabled="disabled">
    <input >

    <input type="checkbox" name="checkbox" value="checkbox1">checkbox1
    <input type="checkbox" name="checkbox" value="checkbox2">checkbox2
    <input type="checkbox" name="checkbox" value="checkbox3">checkbox3

    <select name="select">
        <option>1</option>
        <option>2</option>
        <option>3</option>
    </select>
</form>
</body>
  • 对照上图理解选择器
//注意: 只要标签内值可以发生改变就会被认为是可选因素 例如: select元素
$("#form1 :enabled") //选取id为from1的表单内所有可用元素 (注意 ":" 前面有一个空格,否则不识别)
//注意: 只有标签内有disabled属性时,该标签才会被选中 
$("#form1 :disabled") //选取id为form1的表单内所有不可用元素 (注意 ":" 前面有一个空格,否则不识别)
//以下一下两种选择方式需要定义click事件 
$("input:checked") // 选取input被选中的元素(特指:单选框,多选框) (注意 ":" 前后没有空格,否则不识别)
$("select option:selected") //选取被选中的option标签

  • 专门为表单而生的选择器
  • 注意: “ : “ 冒号前有空格

$("#form1 :button") //选取所有按钮
$("#form1 :file") //选取所有的上传域

1.5 jQuery中的DOM操作

1.5.1 添加节点

$("div").append("<p></p>"); // 在div标签内部末尾插入一对p标签
$("div").prepend("<p></p>"); // 在div标签内部开头插入一对p标签
$("div").appendTo("<p></p>"); // 将div标签插入到p标签末尾中
$("div").prependTo("<p></p>"); // 将div标签插入到p标签开头中
$("div").after("<p></p>"); // 在div标签后面插入p标签
$("div").insertAfter("<p></p>"); // 在div标签前面插入p标签
$("div").before("<p></p>"); // 在div标签前面插入p标签
$("div").insertefore("<p></p>"); // 在p标签前面插入div标签

1.5.2 删除节点

$("div").remove(); //将div节点从网页上删除
$("div").detach(); //该方法并不会吧div节点的绑定事件与追加数据删除,并可以再次追加回来
$("div").empty(); //清空节点

1.5.3 复制节点

$(this).clone(true); //一般常用this表示当前对象,true则是表示连同元素所绑定的事件一起复制

1.5.4 替换节点

$("p[name*='my']").replaceWith("<span></sapn>"); //替换掉name中有my的p标签为span
$("<span></span>").replaceAll("p"); // 替换掉所有p标签为span

1.5.5 包裹节点

$("ul > li[class*='my']:eq(2)").wrap("<b></b>"); // 将匹配的标签包裹在b标签中
$("ul span[class|=my]").wrapAll("<b></b>"); // 将匹配的所有元素用一个b标签包裹起来 如果元素并不在一起,wrapAll()方法会自动将匹配到的元素聚集到一起,位置则在第一个匹配标签的位置
$("ul li:eq(0)").wrapInner("<b></b>"); // 讲匹配到的元素的子元素(包含文本内容)包裹起来

一.相关知识准备:
1.onchange事件:
此事件会在元素内容发生改变，且失去焦点的时候触发。
浏览器支持度较好。
2.onpropertychange事件:
此事件会在元素内容发生改变时立即触发，即便是通过js改变的内容也会触发此事件。
元素的任何属性改变都会触发该事件，不止是value。
只有IE11以下浏览器支持此事件。
3.oninput事件:
此事件会在value属性值发生改变时触发，通过js改变value属性值不会触发此事件。
只有IE8以上或者谷歌火狐等标准浏览器支持。
